Unas de las primeras cosas que hice fue configurar el entorno de
desarrollo, para esto es esenceial usar un entorno virtual, para
crearlo se usa el comando "python -m venv env_tripod", luego para
activarlo se usa el comando "source env_tripod/Scripts/activate"

Luego copiar el proyecto de github a la carpeta del entorno virtual

(bueno en mi caso, copie primero el proyecto tripodpy y dentro de este
condigue el entorno virtual, luego copie el resto de carpetas dentro del
entorno virtual)

Hay que instalar al pie de la letra como dice en la doc de tripodpy. Usar 
python 3.11.x. Antes estaba usando la version 3.14 me daba errores al compilar
con ninja o nija no me acuerda cuales eran.


Una vez todo listo, el proyecto tambien se guarda como una lib dentro
del venv, por lo que lo podemos usar en culauqier carpeta, siempre y cuando
tengamos el venv activado.

El error Factor is exactly singular es la confirmación matemática de lo que sospechábamos:
 cuando dustpylib intenta evaluar tu gap como un proceso dinámico continuo (es decir, el 
 gas moviéndose activamente en cada micro-fracción de segundo para formar y sostener el 
 hueco), el gradiente de densidad se vuelve tan vertical en algunas celdas que al armar la
  matriz jacobiana $A$ se generan ceros en las diagonales (es decir, bloques de matriz 
  singulares o insolubles). Al colapsar el álgebra matricial, bloquea la integración 
  (dt -> 0) y es por eso que el solver se congela durante horas hasta que tú lo 
  interrumpiste.
